Quick Reference Guide

Power Platform formulas, expressions, and keyboard shortcuts — all in one place

Filtering & Lookup
Filter Formula
Filter(DataSource, Condition)
Returns rows that match the condition. Delegable with SharePoint/Dataverse when using simple equality.
Search Formula
Search(DataSource, SearchTerm, "Col1","Col2")
Text-based partial match across specified columns. Not delegable — limited to 500 rows by default.
LookUp Formula
LookUp(DataSource, Condition, ReturnField)
Returns the first matching record or a specific field value from a record.
SortByColumns Formula
SortByColumns(Source, "Column", SortOrder.Ascending)
Sorts a table by one or more columns. Delegable to SharePoint and Dataverse.
Data Writes
Patch — Update Formula
Patch(DataSource, LookUp(DataSource, ID=varID), {Title: "New"})
Updates a specific record. Use Defaults(DataSource) as the second argument to create a new record.
Patch — Create New Formula
Patch(DataSource, Defaults(DataSource), {Title: "New Item", Status: "Active"})
Creates a new record by using Defaults() as the base record.
Remove Formula
Remove(DataSource, LookUp(DataSource, ID = varID))
Deletes a specific record from a data source. Can also remove multiple records with RemoveIf.
ClearCollect Formula
ClearCollect(colMyData, Filter(SharePointList, Status="Active"))
Clears a collection then loads new data into it. Use in OnStart or OnVisible to cache data locally.
Navigation & Variables
Navigate Formula
Navigate(ScreenName, ScreenTransition.Fade)
Moves to another screen. Use None, Fade, Cover, CoverRight, UnCover, or Slide transitions.
Set (Global Variable) Formula
Set(varMyVar, "Hello World")
Creates or updates a global variable accessible from any screen in the app.
UpdateContext (Local Var) Formula
UpdateContext({locIsLoading: true})
Creates or updates a variable scoped to the current screen only. Prefix with "loc" by convention.
Back Formula
Back(ScreenTransition.UnCover)
Navigates to the previously visited screen, reversing the transition used to arrive.
Logic & Text
If / Switch Formula
If(Condition, TrueResult, FalseResult) Switch(varStatus, "Active","Green", "Inactive","Red", "Grey")
If evaluates a condition; Switch matches a value against multiple cases — cleaner than nested If.
Concatenate / & Formula
"Hello " & User().FullName Concatenate("Ref-", Text(ID, "000"))
The & operator is shorthand for Concatenate. Text() formats numbers/dates as strings.
IsBlank / IsEmpty Formula
If(IsBlank(TextInput1.Text), "Required", "OK") IsEmpty(colMyCollection)
IsBlank checks individual values; IsEmpty checks whether a table/collection has any rows.
Now / Today Formula
Now() // date + time Today() // date only Text(Now(), "dd/mm/yyyy hh:mm")
Returns the current date/time. Wrap with Text() to format for display.
CountIf / Sum Formula
CountIf(SharePointList, Status="Active") Sum(colOrders, OrderTotal)
CountIf counts rows matching a condition; Sum adds a numeric column across a table.
User() Formula
User().Email User().FullName User().Image
Returns properties of the currently signed-in user. Useful for personalisation and filtering user-owned records.
Date & Time Expressions
Format Date Expression
formatDateTime(utcNow(), 'dd/MM/yyyy') formatDateTime(utcNow(), 'yyyy-MM-dd HH:mm')
Formats a UTC timestamp into a readable string. Use single quotes around the format pattern.
Add / Subtract Days Expression
addDays(utcNow(), 7) addDays(utcNow(), -30) addDays(triggerBody()?['DueDate'], -3)
Adds or subtracts a number of days from a timestamp. Works with trigger/action outputs too.
Convert Timezone Expression
convertTimeZone(utcNow(), 'UTC', 'Mountain Standard Time')
Converts a timestamp between time zones. Find zone IDs from the Windows Time Zone list.
String Expressions
concat Expression
concat('Hello ', triggerBody()?['Title'])
Joins two or more strings. Equivalent to using the + operator in other languages.
substring Expression
substring(triggerBody()?['Email'], 0, 5) indexOf(triggerBody()?['Email'], '@')
Extracts a portion of a string. indexOf finds the position of a character.
replace / trim Expression
replace(triggerBody()?['Title'], ' ', '-') trim(triggerBody()?['Notes']) toLower(triggerBody()?['Email'])
replace swaps characters, trim removes leading/trailing spaces, toLower lowercases text.
split Expression
split(triggerBody()?['Tags'], ';') first(split(triggerBody()?['FullName'], ' '))
Splits a string into an array by delimiter. Use first() / last() to grab individual parts.
Array & Object Expressions
triggerBody / outputs Expression
triggerBody()?['FieldName'] outputs('ActionName')?['body']?['value']
The ? (null-safe) operator prevents errors when a property is missing. Always use it on dynamic content.
json / string Expression
json(triggerBody()?['JsonField']) string(variables('myObject'))
json() parses a JSON string into an object; string() serialises an object back to a JSON string.
length / empty Expression
length(body('Get_Items')?['value']) empty(body('Get_Items')?['value'])
length() returns the item count of an array or character count of a string. empty() returns true/false.
Flow Patterns
Condition — Check Null Tip
// In a Condition action, use Expression: empty(triggerBody()?['Approver'])
Use empty() or equals(..., null) in Condition actions to safely test whether a field was filled in.
Do Until — Poll Pattern Tip
// Do Until: @equals(variables('Status'), 'Approved') // Inside: Get Item → Set Variable → Delay 5 min
Do Until loops repeatedly until a condition is true. Always add a Delay action inside to avoid throttling.
Scope — Error Handling Tip
// Try scope: your actions // Catch scope (runAfter = Failed/TimedOut): // Send Email / Log Error
Wrap risky actions in a Scope. Set a second Scope to run on failure for a Try/Catch pattern.
Core DAX Measures
CALCULATE DAX
Sales YTD = CALCULATE( SUM(Sales[Amount]), DATESYTD(Calendar[Date]) )
Modifies the filter context. The most important DAX function — all time intelligence relies on it.
SUMX (Iterator) DAX
Revenue = SUMX( Sales, Sales[Qty] * Sales[UnitPrice] )
Row-by-row evaluation, then aggregation. Use X-functions (SUMX, AVERAGEX, MAXX) when you need row context.
RELATED / RELATEDTABLE DAX
// In Sales table (many side): Category = RELATED(Products[Category]) // In Products table (one side): SalesCount = COUNTROWS(RELATEDTABLE(Sales))
RELATED traverses to the one-side; RELATEDTABLE returns all related rows from the many-side.
DIVIDE (Safe Division) DAX
Margin % = DIVIDE( [Gross Profit], [Revenue], 0 -- alternate result if denominator = 0 )
Always use DIVIDE instead of / to avoid divide-by-zero errors in measures.
ALL / REMOVEFILTERS DAX
% of Total = DIVIDE( [Sales], CALCULATE([Sales], ALL(Sales)) )
ALL removes filters from a table/column. Use inside CALCULATE to compute a grand total ignoring slicers.
Year-over-Year DAX
Sales PY = CALCULATE([Sales], PREVIOUSYEAR(Calendar[Date])) YoY % = DIVIDE([Sales] - [Sales PY], [Sales PY])
PREVIOUSYEAR shifts the date filter back 12 months. Requires a marked Date table in your model.
Power Query (M Language)
Filter Rows M
Table.SelectRows(Source, each [Status] = "Active") Table.SelectRows(Source, each [Amount] > 1000)
Filters rows where the condition is true. The each keyword creates a function over each row.
Add Custom Column M
Table.AddColumn(Source, "FullName", each [FirstName] & " " & [LastName])
Adds a new calculated column. Reference column names in square brackets inside the each expression.
Date Parts M
Date.Year([OrderDate]) Date.Month([OrderDate]) Date.MonthName([OrderDate], "en-GB")
Extract year, month number, or month name from a date column. Used when building a Date dimension table.
OData / REST Filters
Equality & Comparison OData
Status eq 'Active' Amount gt 1000 Amount le 500 ModifiedDate ge '2025-01-01T00:00:00Z'
eq, ne, gt, ge, lt, le — standard OData comparison operators. String values need single quotes.
Contains / StartsWith OData
substringof('keyword', Title) startswith(Title, 'Project')
Text search within a field. Note: substringof is not indexed — use sparingly on large lists.
Person / Lookup Field OData
AssignedToId eq 42 // Get current user ID first: /_api/web/currentuser
Filter by person or lookup column using the numeric ID. Use the REST endpoint to get the current user's ID.
AND / OR OData
Status eq 'Active' and Priority eq 'High' Status eq 'New' or Status eq 'In Progress'
Combine conditions with and / or (lowercase). Use parentheses to control precedence.
Column Calculated Formulas
Days Until Due Calculated
=INT([DueDate]-TODAY())
Number of days remaining until a due date. Returns negative if overdue. Use in a Calculated column.
Status Label Calculated
=IF([Status]="Approved","✅ Done",IF([Status]="Rejected","❌ Rejected","⏳ Pending"))
Nested IF for a friendly status display. Calculated columns don't support lookup to external lists.
Concat Text Columns Calculated
=[FirstName]&" "&[LastName] ="REF-"&TEXT([ID],"0000")
Join text fields with & and string literals. TEXT() formats numbers — useful for reference codes.
Lookup Formulas
XLOOKUP Formula
=XLOOKUP(lookup_val, lookup_array, return_array, "Not Found")
Modern replacement for VLOOKUP. Searches any direction, returns multiple columns, and handles not-found gracefully.
INDEX / MATCH Formula
=INDEX(ReturnRange, MATCH(LookupVal, LookupRange, 0))
The classic flexible lookup. MATCH finds the position; INDEX retrieves the value at that position.
VLOOKUP Formula
=VLOOKUP(A2, Sheet2!$A:$C, 3, FALSE)
Looks up a value in the first column and returns a value from a specified column. Always use FALSE for exact match.
Aggregate & Conditional
SUMIFS / COUNTIFS Formula
=SUMIFS(SumRange, CriteriaRange1, Criteria1, CriteriaRange2, Criteria2) =COUNTIFS(Range1, "Active", Range2, ">"&DATE(2025,1,1))
Multi-condition aggregation. Always Sum/Criteria range pairs after the first argument in SUMIFS.
IFERROR Formula
=IFERROR(VLOOKUP(A2, Data, 2, 0), "Not Found")
Wraps any formula to return a fallback value if an error occurs. Cleaner than nested IF(ISERROR(...)).
TEXT / Date Parts Formula
=TEXT(A2,"dd/mm/yyyy") =TEXT(A2,"mmmm yyyy") =YEAR(A2) & "-Q" & INT((MONTH(A2)-1)/3+1)
TEXT formats a date/number as a string. The last formula builds a "2025-Q2" style quarter label.
UNIQUE / FILTER (Dynamic Arrays) Formula
=UNIQUE(A2:A100) =FILTER(A2:C100, B2:B100="Active", "No results") =SORT(UNIQUE(A2:A100))
Dynamic array functions that spill results into multiple cells automatically. Available in Excel 365 / 2021+.
