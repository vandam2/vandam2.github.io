**PowerApps Modern Controls YAML Formatting Rules:**


**1. Inline `[` and `{` characters must use block scalar**

Any property value starting with `[{`, `[`, or `{` directly after `=` on the same line will cause a YAML mapping/sequence parse error. Always use `|-`:

```yaml

# WRONG

DefaultSelectedItems: =[{Value: "New"}]


# CORRECT

DefaultSelectedItems: |-

 =[{Value: "New"}]

```


**2. No bare comment lines between controls**

`//` comments are valid inside PowerFx block scalars but cannot exist as standalone lines between YAML nodes. Never place `// ── Section Label ──` style dividers between controls at the YAML level — they parse as invalid scalars.


**3. `ThisItem` is only valid inside a gallery template**

Controls outside a gallery (footer bars, screen-level buttons) cannot reference `ThisItem`. Use `ForAll(...As task, task.FieldName)` for batch operations, or store the record in a variable first with `Set(varSomething, ThisItem)` before leaving gallery scope.


**4. `colTaskChanges` vs `colTaskEditChanges`**

Home screen uses `colTaskChanges`. EditProject screen uses `colTaskEditChanges`. These are different collections — mixing them causes silent failures where saves or staged rows never appear.


**5. `Text(Value(ProjectID))` for ProjectID normalization**

SharePoint Number fields formatted as text can return comma-formatted strings like `"1,559"`. Always normalize with `Text(Value(ProjectID))` when building collections from `ACA_ProjectTasks` via `ForAll`. Plain `Text(ProjectID)` preserves the comma and breaks all Filter comparisons.


**6. Negative tempID pattern for staged new rows**

New rows not yet saved to SharePoint get `ID: -(CountRows(collection) + 1)`. Save All detects `task.ID < 0` and uses `Defaults(DataSource)` instead of `LookUp`. Discard uses `RemoveIf(collection, ID < 0)`. Never use `Patch` to datasource in Add New — only stage to collections.


**7. `varTaskNotesDraft` drives notes fields**

Text inputs used with bullet/numbered list toolbars must be driven by a variable (`Value: =varTaskNotesDraft`) with `OnChange: =Set(varTaskNotesDraft, Self.Value)`. The toolbar buttons append to the variable then call `Reset(controlName)` to force the field to re-read the updated variable value.


**8. `DisplayMode` using `|+` block scalar**

Multi-line or conditional `DisplayMode` values need `|+` not inline assignment:

```yaml

# WRONG

DisplayMode: =If(varNumberMode, DisplayMode.Edit, DisplayMode.View)


# CORRECT

DisplayMode: |+

 =If(varNumberMode, DisplayMode.Edit, DisplayMode.View)

```


**9. Reset variables on popup close**

Every popup close path (header X button, Cancel button, Stage/Save button) must reset all popup-related variables: `varShowPopup`, `varIsEditing`, `varEditingRecord`, `varTaskNotesDraft`, `varBulletMode`, `varNumberMode`, `varListCounter`. Missing any one causes stale state when reopening.
